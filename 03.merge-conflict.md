# Merge Conflict

## What is a Merge Conflict?
A merge conflict happens when Git cannot automatically combine changes from two branches.

## Why does it happen?
- Same file edited in different branches
- Git cannot decide which change to keep

## How to resolve Merge Conflict?

### Step 1: Merge branch
git merge branch-name

### Step 2: Open conflicted file
Git will show conflict markers:

<<<<<<< HEAD
Current branch code
=======
Incoming branch code
>>>>>>> branch-name

### Step 3: Edit file manually
Choose correct code and remove markers.

### Step 4: Add resolved file
git add file-name

### Step 5: Commit changes
git commit -m "Resolved merge conflict"

## Conclusion
Merge conflicts are normal in Git and can be fixed by editing conflicting code manually.
